â“ Common Interview Questions & Answers

Q: Why did you choose Flask?

"Flask is lightweight and perfect for demonstrating DevOps concepts. The app could be anythingâ€”Node.js, Go, Javaâ€”the DevOps infrastructure around it would be the same. I chose Flask because it's simple to understand and lets us focus on the infrastructure."

Q: Why 3 replicas in Kubernetes?

"Three replicas provide high availability. If one pod fails, two are still serving traffic. It also allows rolling updatesâ€”during deployment, we always have at least 2 pods running. In production, the replica count would depend on traffic and SLOs."

Q: What happens if the EC2 instance runs out of disk space?

"The disk_check.sh script would detect it and alert. In production, I'd configure Prometheus to scrape node-exporter metrics and set up Alertmanager to send alerts to Slack or PagerDuty when disk usage exceeds a threshold."

Q: How would you handle secrets in production?

"Never in code. For CI/CD, I use GitHub Secrets. For Kubernetes, I'd use Kubernetes Secrets or external secret managers like AWS Secrets Manager or HashiCorp Vault. For Terraform, I'd use environment variables or AWS IAM roles."

Q: How would you scale this for more traffic?

"In Kubernetes, I'd add Horizontal Pod Autoscaler to automatically scale replicas based on CPU or custom metrics. For the EC2 approach, I'd use an Auto Scaling Group with a load balancer. Terraform can provision both."

Q: What would you add for a production system?

"Logging aggregation with ELK or Loki, distributed tracing with Jaeger, HTTPS with cert-manager, network policies, pod security policies, proper RBAC, backup strategies, and disaster recovery procedures."
